Funktionen

Was Schießkladde kann

Alle Kernfunktionen einer modernen Standverwaltung — ohne Bloat, ohne Cloud.

📊

Echtzeit-Dashboard

Live-Übersicht aller Bahnen: frei, belegt, Wartung oder defekt. Aktuell ohne Seitenreload.

📋

Schussbuch §24 AWaffV

Automatische Erstellung bei Bahnenfreigabe. Einträge sind unveränderlich; Stornierung nur mit Begründung und Protokoll.

🎽

Ausrüstungsverwaltung

Verleih und Rückgabe von Waffen, Gehörschutz, Schutzbrillen u. v. m. — vollständig dokumentiert.

📶

RFID-Check-in/out

Schütze hält Karte ans Lesegerät → Bahn wird freigegeben. Keine Aufsicht nötig für den Abmeldevorgang.

🔐

Rollenverwaltung

Admin, Aufsicht, Betrachter — mit fein­granulierten Berechtigungen, die der Vorstand selbst konfiguriert.

⚡

Hardware-Integration

ANEL NET-PwrCtrl schaltet Bahnenbeleuchtung oder -strom automatisch — per HTTP oder UDP, pro Bahn konfigurierbar.

📈

Statistiken & Export

Auslastungsanalysen, Belegungsgrafiken, Schussbuch als Excel-Datei exportierbar.

❓

Eingebautes Hilfesystem

Kontextsensitive Anleitungen direkt in der App — kein externes Wiki, keine PDF-Bedienungsanleitung nötig.

Rechtliches

Compliance eingebaut

Kein Nachdenken nötig — die App hält die gesetzlichen Vorgaben automatisch ein.

AWaffV §24

Schusswaffen­rechtliches Schussbuch

  • Schussbucheinträge werden automatisch beim Verlassen der Bahn erzeugt
  • Keine physische Löschung — nur Stornierung mit Begründung und Benutzerprotokoll
  • Mindestaufbewahrung 5 Jahre (konfigurierbar, nie unterschreitbar)
  • Vollständige Erfassung: Name, Geburtsdatum, Datum, Uhrzeit, Waffe, Kaliber, Bahn
  • Anonymisierung nach konfigurierbarer Frist mit Admin-Bestätigung
DSGVO

Datenschutz­grundverordnung

  • Ausschließlich On-Premise — keine Daten verlassen das Vereinsnetz
  • Kein Cloud-Dienst, keine externen Analysetools, kein Tracking
  • Rollenbasierter Zugriff — Daten nur für berechtigte Benutzer sichtbar
  • Passwörter bcrypt-gehasht, keine Klartextspeicherung
  • Vollständiges Aktivitätsprotokoll aller Admin- und Aufsichtshandlungen
  • Konfigurierbare Lösch- und Anonymisierungsfristen

Für technisch Versierte

Tech Stack

Next.js 16 React 19 TypeScript 5 TailwindCSS 4 shadcn/ui Prisma 7 PostgreSQL next-auth v5 Zod 4 React Hook Form SWR Recharts ExcelJS Playwright E2E

CI/CD Pipeline — GitHub Actions (4 Workflows)

git push → Security Gate → DSGVO Gate → Quality Gate → E2E Tests → GitHub Release → Deploy

Security Gate

npm audit (HIGH/CRITICAL), Semgrep SAST, TruffleHog Secret-Scan — blockiert bei Fund

DSGVO Gate

Bearer DSGVO-Scan — erkennt PII-Probleme bereits im Code-Review

Quality Gate

TypeScript strict, ESLint, Lizenz-Check (blockiert GPL/AGPL), Lighthouse CI

E2E Tests

Playwright gegen echte PostgreSQL-Testdatenbank — 150 Tests, 13 Spec-Dateien

GitHub Release

Vorkompiliertes Next.js-Standalone-Bundle als schieskladde.tar.gz — kein Node/npm auf dem Server

Deploy

update.sh lädt Tarball, extrahiert, führt Prisma-Migra­tion aus, startet systemd-Service neu

Deployment-Architektur

Laufzeitumgebung

Proxmox LXC (Debian 12)
kein Docker, kein Kubernetes

Ressourcen

2 vCPU · 1 GB RAM · 8 GB Disk
läuft auf Standard-Vereinshardware

Prozess-Manager

systemd-Service
Autostart, Crash-Restart, Logging via journalctl

Netzwerk

VLAN-isoliert
kein Internet-Exposure, nur im Vereinsnetz erreichbar

Datenbank

PostgreSQL lokal
Prisma 7 mit WASM-Engine + pg-Adapter

Updates

Ein Script-Aufruf
